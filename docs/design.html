A. このシステムは何か

AIを一切使わない、軽量な開発者向けユーティリティAPIを1本だけ作り、RapidAPI（APIのマーケットプレイス）で従量課金で売る仕組みです。 課金・利用者管理・APIキー発行は RapidAPI が肩代わりするので、サーバーは「正しいRapidAPI経由か？」だけ確認すれば済みます。

✓
低コスト構造：CPUをほとんど使わない処理ばかり（QR・ハッシュ・変換・画像SVG等）。AI不使用なのでトークン代ゼロ。サーバーは無料〜数百円/月。

登場人物は4つだけ

① あなた

コードを GitHub に置く。手を動かすのは最初の1回だけ。

② GitHub → Render

push すると Render（API本体が動くサーバー）へ自動デプロイ。

③ RapidAPI

販売窓口。登録・課金・回数制限・鍵発行を全部代行。

④ 利用者

世界中の開発者。RapidAPI経由でAPIを叩く（彼らが支払う）。
◎
1コード → 2出品：コードは1本のまま、RapidAPIの出品を「日本向け（日本語・円圏）」と「グローバル（英語・ドル圏）」の2枚にして同時展開。追加開発コストはほぼゼロ。

リクエストの流れ（1回の呼び出し）

1

RapidAPI ゲートウェイ

利用者の残回数チェック → 課金 +1 → 秘密ヘッダ付きで転送

2

onRequest フック（認証）

X-RapidAPI-Proxy-Secret を検証。合致しなければ 401。

3

レート制限

ティア別の「1分あたり上限」を確認（瞬間スパイク防止）。

4

ルート処理 → 応答

例：QRを生成して返す。onResponse で利用メーターに +1。

C. 認証の3経路（src/auth.ts）

優先経路判定ティア
1RapidAPI経由X-RapidAPI-Proxy-Secret 一致pro
2直接APIキーX-API-Key が登録済みキー次第
3匿名(dev)ALLOW_ANONYMOUS=truefree
!
本番では「1」が主役。RapidAPI以外の直アクセスは秘密鍵を知らず弾かれるので、課金を迂回されない設計です。

ティアとレート制限

ティア1分上限(既定)用途
free30無料プラン／お試し
pro300有料プラン（RapidAPI既定）
ultra3000大口・上位プラン

環境変数（Renderダッシュボードで設定）

変数説明
RAPIDAPI_PROXY_SECRET本番で必須。RapidAPIのSecurity設定の値
API_KEYS直販用。key:tier をカンマ区切り
ALLOW_ANONYMOUSdevのみ true。本番は false
RATE_LIMIT_*FREE/PRO/ULTRA の1分上限を上書き
🔒
秘密はGitに置かない。render.yaml は sync:false にしてあり、実値はダッシュボードのみで入力します。

F. 初回セットアップ（約1時間・一回限り）

1

リポジトリを取得

受け取ったGitHubリポジトリをクローン

TERMINAL
git clone <your-repo-url>
cd toolbelt-api
npm install
2

動作確認

ローカルで立ち上げてドキュメントを確認

TERMINAL
npm run dev
# ブラウザで http://localhost:3000/docs
3

Render でデプロイ

render.com で New → Blueprint → GitHub接続 → render.yaml を自動検出 → デプロイ。ダッシュボードで環境変数を設定。

4

RapidAPI に出品

① Render の URL を Base URL に設定 ② Security で Proxy Secret を生成し、その値を Render の RAPIDAPI_PROXY_SECRET に登録 ③ OpenAPI を取り込み ④ 無料/Pro/Ultra の価格を設定

5

入金口座を接続

RapidAPI に Stripe 等の受取口座を接続。

6

（任意）2枚目の出品

同じ Base URL で英語版の出品を複製 → グローバル同時展開。
